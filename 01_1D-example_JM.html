Introduction

In this vignette we are going to see how to fit an SPDE to one-dimensional point data, i.e.Â data that consist of the points at which things are located, not the number of points in some area. This is essentially the outcome of a Poisson process in one dimension.

Fiting the model

This proceeds in four stages:

  1. Define the mesh (essentially the set of locations where knots are to be placed).
  2. Define SPDE part of model on the mesh (indicates the autocorrelation structure).
  3. Define model formula, including the SPDE object
  4. Fit the model to the data

Stage 1: Build a 1D mesh

x <- seq(0, 55, length.out = 50)
mesh1D <- inla.mesh.1d(loc=x, boundary = "free")

The locations loc of the knots and the type of boundary condition to be implemented. The boundary is optional, but if necessary could be specified in several different ways (and it can even be different at the two extremes of the mesh).

Stage 2: Make a 1D SPDE model

matern <- inla.spde2.pcmatern(mesh1D, prior.range = c(150, 0.75), prior.sigma = c(0.1, 0.75))

The generic function inla.spde2.generic creates an inla.spde2 object, describing the internal structure of an spde2 model. The pcmatern version of this employs a Matern model with a PC prior for the parameters. The Matern model expresses the covariance between two points as a decreasing function of their distance. The two parameters range and sigma are assigned priors in the typical INLA fashion. For example, the prior for a parameter \(\theta\) is given by c(theta0,Ptheta)=\((\theta_0,P_\theta)\), such that \(P(\theta<\theta_0)=P_\theta\).

Stage 3: Define model formula, including intercept

mdl <- x ~ spde1D(x, model = matern) + Intercept(1)

Note that this is just an assignment to the name mdl of a model formula, where the autocorrelation in the variable x is captured solely by the one-dimensional spde model. Presumably, the previously defined model matern could be used for modelling the autocorrelation in a variable other than x.

Stage 4: Fit the model

Here, we want to fit to the actual points, and the inlabru function that does this is called lgcp (for âLog Gaussian Cox Processâ). The ips and domain parameters are optional, one or the other can be used. Here are two ways of doing the same thing:

fit.spde <- lgcp(mdl, pts2, ips = ipoints(c(0, 55), 50, name = "x"))
fita.spde <- lgcp(mdl, pts2, domain = list(x = mesh1D))

SPDE parameters

We can look at the posterior distributions of the parameters of the SPDE using the function spde.posterior. It returns x and y values for a plot of the posterior PDF in a data frame, which can be printed using the plot function. To see the PDF for the range parameter, for example:

post.range <- spde.posterior(fit.spde, name = "spde1D", what = "range")
plot(post.range)

Look at the help file for spde.posterior and then plot the posterior for the log of the SPDE range parameter, the SPDE variance and/or log of the variance, and for the Matern covariance function. Predicting intensity

We can also now predict on any scale we want. For example, to predict on the âresponseâ scale (i.e.Â the intensity function \(\lambda(s)\)), we call predict thus:

pred_spde <- predict(fit.spde, predf, ~ exp(spde1D + Intercept))

while to predict on the linear predictor scale (i.e.Â that of the log intensity, \(\log(\lambda(s))\)), we call predict thus:

How does this compare with the underlying intensity function that generated the data? The function lambda2_1D( ) in the dataset Poisson2_1D calculates the true intensity that was used in simulating these data. In order to plot this, we make a data frame with x- and y-coordinates giving the true intensity function, \(\lambda(s)\). We use lots of x-values to get a nice smooth plot (150 values).

xs <- seq(0, 55, length = 150)
true.lambda <- data.frame(x = xs, y = lambda2_1D(xs))

Plot the fitted and true intensity functions:

Goodness-of-Fit

We can look at the goodness-of-fit of the mode using the inlabru function bincount( ), which plots the 95% credible intervals in a specified set of bins along the x-axis together with the observed count in each bin: The credible intervals are shown as red rectangles, the mean fitted value as a short horizontal blue line, and the observed data as black points:

bc <- bincount(
  result = fit.spde,
  observations = pts2,
  breaks = seq(0, max(pts2), length = 12),
  predictor = x ~ exp(spde1D + Intercept)
)

Estimating Abundance

Abundance is the integral of the intensity over space. We estimate it by integrating the predicted intensity over x. Integration is done by adding up the intensity at locations x weighted by a particular weight. The locations x and their weights are constructed using the ipoints function (see also use if this function as an option for defining the integration points in the fit.spde step)

ips <- ipoints(c(0, 55), 100, name = "x")
head(ips)
#>       x weight
#> 1 0.275   0.55
#> 2 0.825   0.55
#> 3 1.375   0.55
#> 4 1.925   0.55
#> 5 2.475   0.55
#> 6 3.025   0.55
Lambda <- predict(fit.spde, ips, ~ sum(weight * exp(spde1D + Intercept)))

This approximates the integral over space as a sum of finite areas, using regularly-spaced integration points. Presumably, we could define irregularly spaced integration points via the function ipoints. You can look at the abundance estimate by typing

Lambda
#>       mean       sd   q0.025   median   q0.975     smin     smax         cv      var
#> 1 129.8393 12.00306 109.7047 128.2675 153.8193 103.7467 155.7596 0.09244546 144.0734
  • mean is the posterior mean abundance.
  • sd is the estimated standard error of the posterior of the abundance.
  • cv is its estimated coefficient of variation (stander error divided by mean).
  • q0.025 and q0.975 are the 95% credible interval bounds.
  • q0.5 is the posterior median abundance

But it is not quite that simple! The above posterior for abundance takes account only of the variance due to us not knowing the parameters of the intensity function. It neglects the variance in the number of point locations, given the intensity function. In other words, the uncertainty in the abundance estimate represents the âconfidence intervalâ, not the âprediction intervalâ in the fitted model. To include this we need to modify predict( ) as follows:

Nest <- predict(

    N = 50:250,
    dpois = dpois(50:250,
